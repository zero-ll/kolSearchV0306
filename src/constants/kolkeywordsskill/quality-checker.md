# 子 SKILL 3：质量检查层（反问检查 + Strategy Explanation）

## 在整个流程中的位置

- **位置**：流程第三步（最后一步），由主 SKILL（SKILL.md）在子 SKILL 2 完成后调用
- **输入**：子 SKILL 2（keyword-generator.md）输出的候选关键词草稿，格式如下：

```json
{
  "keyword_strategy_groups_draft": [
    {
      "strategy_type": "",
      "keyword_group_name": "",
      "keyword_list": [],
      "recommended_creator_tiers": []
    }
  ]
}
```

- **输出**：通过三问检查的最终关键词、每组的 strategy_explanation、tags，以及填充完整的 ai_reasoning.reflection_check，格式如下：

```json
{
  "keyword_strategy_groups_final": [
    {
      "strategy_type": "",
      "keyword_group_name": "",
      "keyword_list": [],
      "strategy_explanation": "",
      "tags": {
        "创作者类型": [],
        "内容形态": [],
        "适合层级": []
      },
      "recommended_creator_tiers": []
    }
  ],
  "reflection_check": [
    {
      "keyword": "",
      "q1_search_result_quality": "",
      "q2_creator_language_match": "",
      "q3_tokenization_risk": "",
      "verdict": ""
    }
  ]
}
```

完成后将此输出返回给主 SKILL，由主 SKILL 汇总组装成最终 JSON。

---

## 反问检查层：输出前强制执行，不通过不得输出

所有关键词生成完毕后，在确认最终关键词之前，必须对每一条候选关键词逐条回答以下三个问题。三个问题必须全部回答，不允许跳过或模糊作答。

---

### 问题 1：如果在 YouTube 搜索这条词，排名靠前的视频，主要属于目标垂类，还是属于其他内容生态？

判断依据：基于对 YouTube 内容生态的认知，这条词的每个组成词，是否在目标垂类以外的内容生态中大量出现？如果是，排名靠前的结果大概率被无关内容占据。

**输出到**：`reflection_check[n].q1_search_result_quality`

---

### 问题 2：这条词，是目标创作者会真实用在自己视频标题里的表达吗？

判断依据：这条词是从产品视角生成的消费者搜索词，还是从创作者视角生成的内容描述词？消费者搜索词在 YouTube 上不是创作者的标题语言，命中的视频和创作者会大量偏离目标。

**输出到**：`reflection_check[n].q2_creator_language_match`

---

### 问题 3：这条词是否会被 YouTube 的拆词匹配机制大量触发到无关内容？

判断依据：当 YouTube 把这条词拆开，每个单词单独匹配时，是否会触发大量与目标垂类无关的高流量内容？

**输出到**：`reflection_check[n].q3_tokenization_risk`

---

### 处理规则

- 三个问题全部通过 → 该词可以输出，`verdict` 填写"通过"
- 任何一个问题答案为否或不确定 → 该词直接丢弃，`verdict` 填写"丢弃并替换"，在原策略的约束框架下重新生成替换词，替换词必须重新通过三问检查
- 不允许对不通过的词做修改后直接输出，必须重新生成
- 如果某个策略组经过多次重新生成后仍无法通过三问检查，在该组的 `strategy_explanation` 中主动说明该策略在当前产品情况下召回效果有限，并给出使用建议

**reflection_check 只记录通过的词**，所有通过的词必须与 `keyword_strategy_groups_final` 里的 keyword_list 完全对应，不多不少。

---

## Strategy Explanation Rules

每组必须输出 `strategy_explanation`，采用**两层结构**写作，必须完整覆盖 6 个维度，缺一不可。

---

### 两层结构要求

**第一层：结论句（1句话）**
用一句话点明这组词的核心用途——"用什么词找什么人"。
这句话是业务扫一眼就能抓住重点的锚点，必须简洁、直接、可感知。

示例风格：
- "用品类核心词加测评格式词，精准找到长期产出造浪机器械评测的垂直创作者。"
- "用竞品品牌词加 vs / alternative，定位已经制作过同类产品内容、受众有购买意图的创作者。"
- "用真实生活场景词加日常记录格式词，找到真实拥有庭院泳池环境的生活方式类博主。"

**第二层：策略细节（2~3句话）**
按以下**固定语序**展开，将剩余 5 个维度自然串联成流畅段落：

> ① 关键词结构逻辑（组合公式是什么）
> → ② 为什么创作者会用这些词（从 YouTube 内容标题动机解释，不从产品逻辑解释）
> → ③ 命中频道类型（2~3种具体频道）
> → ④ 搜索结果形态（视频长什么样）
> → ⑤ 适合层级及原因（为什么这类词适合该层级）

---

### 6 个维度说明

| 维度 | 内容要求 |
|------|---------|
| ① 关键词结构逻辑 | 说清楚组合公式，如"品类词 + review"，不能只说"围绕 XX 词" |
| ② 创作者为何用这些词 | 从内容标题动机解释——创作者用这类词是为了获得什么搜索流量，禁止从品牌卖点角度解释 |
| ③ 命中频道类型 | 列举 2~3 种具体频道类型，如"器械测评频道 / 家庭泳池生活频道 / 游泳训练频道" |
| ④ 搜索结果形态 | 说明搜索后结果页大概出现什么视频，如"开箱测评 / 庭院 vlog / setup 教程" |
| ⑤ 适合层级及原因 | 不能只写层级名，必须解释为什么这组词匹配该层级的创作者规模和内容特征 |
| ⑥ 策略解决的搜索问题 | 融入第一层结论句中体现，如"精准锁定垂类""找真实场景创作者""捕捉竞品已覆盖红人" |

---

### 完整示例（competitor_capture 组）

> **结论句：** 用竞品品牌词加 vs / alternative / worth it，定位已制作过同类产品内容、受众处于主动比较阶段的创作者。
>
> **策略细节：** 关键词结构为「竞品品牌（Endless Pools / Badu）+ review / vs / alternative」，创作者选择制作竞品评测是因为这类词自带搜索流量——用户在购买决策阶段会主动搜索品牌对比，因此这类视频既有流量又有转化价值，频道类型通常是泳池设备测评频道、高端家用健身器材频道、swim spa 对比频道，搜索结果以横向对比视频和"值不值得买"评测为主。适合腰部红人，因为能持续产出竞品对比内容的创作者通常有稳定的垂类受众，内容专业度和受众购买意图都更强。

注意：示例仅供理解写法，不要照抄，名称和内容需根据实际项目动态生成。

---

### 写作禁止项

- 禁止第一层和第二层写成同质内容，第一层是结论，第二层是支撑
- 禁止维度② 从产品卖点角度解释（例如"因为这个功能很重要"）
- 禁止出现无实质内容的结论句，包括："转化价值高" / "适合推广曝光" / "粉丝目的性强" / 任何没有原因支撑的结论
- 禁止写成所有组通用的模板表达，必须结合该组实际关键词内容
- 禁止将 6 个维度单独加标题列出，必须写成自然连贯的段落

---

### 长度要求

- 总长度控制在 **150~250 个中文字符**
- 第一层结论句：20~40 字
- 第二层策略细节：120~210 字
- 中文输出，语气专业、直接，面向红人营销经验较少的业务人员

---

## tags 字段填充规则

`tags` 字段必须从 `strategy_explanation` 内容中提取，三个子字段规则如下：

- `创作者类型`：中文，2~3 个，直接对应 strategy_explanation 第二段中"命中频道类型"的内容
- `内容形态`：英文原词，2~3 个，使用 review / vlog / how to / setup / comparison / day in the life / unboxing / routine / essentials 等内容表达词
- `适合层级`：与 `recommended_creator_tiers` 保持完全一致，不单独维护两套

---

## 输出说明

所有关键词通过三问检查、strategy_explanation 撰写完毕后，输出以下完整结构，返回给主 SKILL：

```json
{
  "keyword_strategy_groups_final": [
    {
      "strategy_type": "（固定枚举值，与子SKILL2保持一致）",
      "keyword_group_name": "（与子SKILL2保持一致，除非因词替换后需要调整）",
      "keyword_list": ["通过三问检查的最终英文关键词"],
      "strategy_explanation": "（两层结构，150~250中文字符）",
      "tags": {
        "创作者类型": ["中文频道类型1", "中文频道类型2"],
        "内容形态": ["英文内容格式词1", "英文内容格式词2"],
        "适合层级": ["头部", "腰部"]
      },
      "recommended_creator_tiers": ["头部", "腰部"]
    }
  ],
  "reflection_check": [
    {
      "keyword": "被检查的关键词原文",
      "q1_search_result_quality": "搜索结果质量判断：该词搜索后排名靠前的内容主要属于什么生态",
      "q2_creator_language_match": "创作者语言匹配判断：该词是否符合创作者真实标题表达习惯",
      "q3_tokenization_risk": "拆词风险判断：该词被YouTube拆词后是否会触发大量无关内容",
      "verdict": "通过"
    }
  ]
}
```

注意：
- `reflection_check` 只记录最终输出的通过词，数量必须与所有 `keyword_list` 中的词总数完全一致
- 被丢弃的词不记录在 `reflection_check` 中
- `keyword_group_name` 通常与子 SKILL 2 保持一致，仅当关键词全部被替换导致组名已不准确时才重新生成
